## 📓 Задание 1 — Личный дневник

### Что реализовано

Личный дневник с хранением записей во **внутреннем (приватном) хранилище** приложения.  
Каждая запись — отдельный `.txt`-файл в папке `filesDir/diary/`.

### Как это работает

**Создание записи:**
1. Пользователь нажимает FAB-кнопку `+`
2. Открывается экран редактирования с необязательным заголовком и большим полем для текста
3. При нажатии «Сохранить» файл записывается в `context.filesDir/diary/`

**Имя файла** формируется автоматически:
```
20250420_143022_Мой заголовок.txt
  ↑ дата    ↑ время  ↑ заголовок (если указан)
```

**Формат содержимого файла:**
```
Мой заголовок

Текст записи начинается здесь.
Может занимать несколько строк...
```

**Список записей** (LazyColumn) обновляется **без пересканирования папки**:
- При запуске приложения — один полный скан `filesDir/diary/`
- При добавлении — новая запись вставляется в начало списка в памяти
- При удалении — запись удаляется из списка по имени файла

**Удаление** — долгое нажатие на карточку → контекстное меню → «Удалить».

### Ключевые API

| API | Использование |
|-----|--------------|
| `context.filesDir` | Путь к приватному хранилищу |
| `File.writeText()` | Сохранение записи |
| `File.readText()` | Чтение для редактирования |
| `File.listFiles()` | Сканирование при старте |

## 📷 Задание 2 — Фотогалерея с камерой

### Что реализовано

Личная фотогалерея с возможностью делать снимки прямо из приложения.  
Фотографии хранятся в **приватном внешнем хранилище** приложения —  
`getExternalFilesDir(Environment.DIRECTORY_PICTURES)`.

### Как это работает

**Съёмка фото:**
1. Нажатие FAB-кнопки с иконкой камеры
2. Запрос разрешения `CAMERA` (через `ActivityResultContracts.RequestPermission`)
3. Генерация временного файла с именем `IMG_yyyyMMdd_HHmmss.jpg`
4. Получение `content://` URI через `FileProvider` (без прямой передачи пути файла)
5. Запуск системной камеры через `ActivityResultContracts.TakePicture`
6. После успешной съёмки файл добавляется в начало списка галереи

**Отображение** — `LazyVerticalGrid` с тремя колонками, квадратные превью через Coil.

**Нажатие на фото** — открывается экран полноэкранного просмотра.

**Долгое нажатие** — контекстное меню с пунктами «Экспорт в галерею» и «Удалить».

### FileProvider — безопасная передача файлов камере

Напрямую передавать путь `file://` камере **запрещено** начиная с Android 7.  
Вместо этого используется `FileProvider`:

```xml
<!-- AndroidManifest.xml -->
<provider
    android:name="androidx.core.content.FileProvider"
    android:authorities="${applicationId}.fileprovider"
    android:grantUriPermissions="true">
    <meta-data
        android:resource="@xml/file_paths" />
</provider>
```

```xml
<!-- res/xml/file_paths.xml -->
<paths>
    <external-files-path name="external_pictures" path="Pictures/" />
</paths>
```

## 🖼 Задание 3 — Экспорт фото в системную галерею

### Что реализовано

Кнопка экспорта на экране просмотра фото сохраняет копию снимка  
в **системную галерею** устройства через `MediaStore` — без запроса разрешений  
(на Android 10+ они не нужны для записи в `MediaStore`).

### Как это работает (паттерн IS_PENDING)

Запись через `IS_PENDING` гарантирует атомарность: файл появится в галерее  
только после полной записи, не в виде «битого» фото.

```
1. Создать запись в MediaStore с IS_PENDING = 1
         ↓
2. Открыть OutputStream и скопировать байты из исходного файла
         ↓
3. Установить IS_PENDING = 0  →  фото появляется в галерее
```

После экспорта — `Snackbar` с сообщением «Фото добавлено в галерею».  
В случае ошибки — `Snackbar` с описанием проблемы.

### Куда сохраняется

```
/storage/emulated/0/Pictures/Module5/IMG_yyyyMMdd_HHmmss.jpg
```

## ✅ Задание 4 — Список дел (TodoList)

### Что реализовано

Полноценное CRUD-приложение со следующими возможностями:
- Добавление, редактирование и удаление задач через **Room Database**
- **Одноразовый импорт** задач из JSON-файла при первом запуске
- **Цветовая подсветка** выполненных задач (настройка через **DataStore**)
- Переключатель цвета прямо в `TopAppBar`

---

### Room Database

Задачи хранятся в SQLite-базе данных через Room ORM.

**Схема таблицы `todos`:**

| Столбец | Тип | Описание |
|---------|-----|----------|
| `id` | INTEGER PK AUTOINCREMENT | Уникальный идентификатор |
| `title` | TEXT | Название задачи |
| `description` | TEXT | Описание (необязательно) |
| `isCompleted` | INTEGER (0/1) | Статус выполнения |
| `createdAt` | INTEGER | Timestamp создания (мс) |

Список задач отображается в порядке убывания `createdAt` — новые задачи вверху.

---

### Одноразовый импорт из JSON

При **первом запуске** (когда база пустая) приложение автоматически импортирует  
задачи из файла `res/raw/sample_todos.json`:

```json
[
  { "title": "Изучить Jetpack Compose",   "isCompleted": true  },
  { "title": "Реализовать Room Database", "isCompleted": true  },
  { "title": "Настроить DataStore",       "isCompleted": false },
  ...
]
```

Флаг `json_imported` сохраняется в **DataStore** — повторный импорт не произойдёт  
даже после перезапуска приложения.

---

### DataStore — настройка цвета выполненных задач

Заменяет устаревший `SharedPreferences`. Хранит два ключа:

```
json_imported       → Boolean  (был ли импортирован JSON)
use_completed_color → Boolean  (подсвечивать ли выполненные задачи)
```

**Переключатель** находится в правой части `TopAppBar` на экране списка.  
Настройка сохраняется мгновенно и переживает перезапуск приложения.

**Выполненная задача с включённым цветом:**
```
┌──────────────────────────────────┐  ← зелёный фон (#E8F5E9)
│ ☑  ~~Изучить Jetpack Compose~~   │
│    Пройти официальный курс...    │
└──────────────────────────────────┘
```

**Выполненная задача с выключённым цветом:**
```
┌──────────────────────────────────┐  ← стандартный фон surfaceVariant
│ ☑  ~~Изучить Jetpack Compose~~   │
└──────────────────────────────────┘
```

## 🏗 Архитектура проекта

```
com.example.module5/
│
├── Module5Application.kt        ← Application-класс, DI вручную
├── MainActivity.kt
│
├── navigation/
│   └── NavGraph.kt              ← Единый граф навигации, activity-scoped VM
│
├── home/
│   └── HomeScreen.kt            ← Главный экран (3 карточки)
│
├── ui/theme/                    ← Material 3 тема
│
├── diary/                       ── ЗАДАНИЕ 1 ──────────────────────
│   ├── DiaryEntry.kt            ← Data class
│   ├── DiaryViewModel.kt        ← Файловые операции + StateFlow
│   ├── DiaryListScreen.kt       ← LazyColumn + FAB + long-press меню
│   └── DiaryEditScreen.kt       ← Создание / редактирование записи
│
├── gallery/                     ── ЗАДАНИЯ 2-3 ─────────────────────
│   ├── GalleryViewModel.kt      ← Камера, MediaStore, SharedFlow событий
│   ├── GalleryScreen.kt         ← LazyVerticalGrid (3 колонки)
│   └── PhotoViewScreen.kt       ← Полноэкранный просмотр + экспорт
│
└── todo/                        ── ЗАДАНИЕ 4 ──────────────────────
    ├── data/
    │   ├── local/               ← Room: Entity, DAO, Database
    │   ├── repository/          ← TodoRepositoryImpl
    │   ├── preferences/         ← DataStore-обёртка
    │   └── model/               ← TodoJson (для Gson)
    ├── domain/
    │   ├── model/               ← Todo (доменная модель)
    │   ├── repository/          ← TodoRepository (интерфейс)
    │   └── usecase/             ← 5 use cases:
    │       ├── GetTodosUseCase         возвращает Flow<List<Todo>>
    │       ├── AddTodoUseCase
    │       ├── UpdateTodoUseCase
    │       ├── DeleteTodoUseCase
    │       └── ImportTodosFromJsonUseCase
    └── presentation/
        ├── TodoViewModel.kt     ← StateFlow + SharedFlow событий
        ├── TodoListScreen.kt    ← LazyColumn + Switch в TopAppBar
        └── TodoEditScreen.kt    ← Форма создания / редактирования
```

### Почему ViewModels привязаны к Activity

Все три ViewModel (`DiaryViewModel`, `GalleryViewModel`, `TodoViewModel`) создаются  
с `viewModelStoreOwner = activity`. Это позволяет экранам списка и редактирования  
**разделять один и тот же экземпляр ViewModel**, избегая рассинхронизации данных  
при навигации «назад».
